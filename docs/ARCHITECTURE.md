# v810-hdl Architecture

> Status: skeleton. Refined as specification work progresses.

## 1. The V810 at a glance

- 32-bit RISC, little-endian
- 32 general-purpose registers (r0–r31), r0 hardwired to zero
- 32-bit linear address space
- **5-stage pipeline**: IF, ID, EX, MEM, WB
- **Hardware interlocks** for RAW hazards (no software NOPs required)
- **IEEE 754 single-precision FPU** on-die
- **Bit-string instructions** (multi-cycle, interruptible)
- Two privilege levels (system, user)
- Vectored interrupt model

## 2. Top-level module interface (proposed)

```
module v810 (
    input  logic        clk,
    input  logic        rst_n,

    // Instruction memory port
    output logic [31:0] imem_addr,
    output logic        imem_req,
    input  logic [31:0] imem_rdata,
    input  logic        imem_ack,

    // Data memory port (byte-addressable, with byte-enable)
    output logic [31:0] dmem_addr,
    output logic        dmem_req,
    output logic        dmem_we,
    output logic [3:0]  dmem_be,
    output logic [31:0] dmem_wdata,
    input  logic [31:0] dmem_rdata,
    input  logic        dmem_ack,

    // Interrupts
    input  logic [4:0]  int_level,     // 0 = none, 1..15 = priority
    input  logic        nmi,

    // Halt / debug (optional)
    output logic        halted
);
```

Two separate memory ports (Harvard-ish) let integrators multiplex or unify on the outside as they prefer. Internal caches are **not** part of v1; the PC-FX's uPD70732 had a small cache but it is not critical for correctness.

## 3. Pipeline stages

- **IF**: fetch instruction at PC. Handles branch misprediction redirect.
- **ID**: decode, read register file, resolve operands, sign/zero-extend immediates.
- **EX**: ALU operations, branch resolution, FPU dispatch (multi-cycle path).
- **MEM**: data memory access.
- **WB**: register file write.

Interlocks stall IF/ID when downstream EX/MEM has not yet produced a required operand.

## 4. Instruction categories

| Category | Examples | Notes |
|---|---|---|
| Register-register | `MOV`, `ADD`, `SUB`, `AND`, `OR`, `XOR` | 1-cycle EX |
| Immediate | `MOVHI`, `ADDI`, `ANDI` | 16-bit imm, sign/zero extended |
| Load/store | `LD.B`, `LD.H`, `LD.W`, `ST.B`, `ST.H`, `ST.W` | Displacement addressing |
| Branch | `BCOND`, `JMP`, `JR`, `JAL` | Conditional + unconditional |
| Bit-string | `SCH0BSU`, `SCH1BSD`, `ORBSU`, etc. | Multi-cycle, interruptible |
| FPU | `ADDF.S`, `SUBF.S`, `MULF.S`, `DIVF.S`, `CVT.WS`, `CVT.SW` | Multi-cycle EX |
| System | `RETI`, `TRAP`, `HALT`, `LDSR`, `STSR` | Privilege-sensitive |
| Special | `CAXI`, `SETF`, `CLI`, `SEI` | Atomics, flags, interrupt control |

## 6. FPU

- IEEE 754 single-precision
- Opcodes: ADDF.S, SUBF.S, MULF.S, DIVF.S, CMPF.S, CVT.WS, CVT.SW, TRNC.SW
- Exception conditions: invalid, divide-by-zero, overflow, underflow, inexact, reserved operand
- Implementation strategy: iterative divide, single-cycle add/sub/mul with DSP blocks if available but fallback LUT-only path exists
